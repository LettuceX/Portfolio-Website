Control System for Battery Testing (MITS 11)

Role: Lead system designer & full stack developer & UX designer Team: 4 People
Architecture: Event-Driven, Client-Server, Cloud-Native
Key Technologies: Apache Kafka, TCP/IP, WebSockets, SQL/Cloud DB, Python/C# (Backend), React/Vue (Frontend)

Project Description

This project involved re-architecting a legacy desktop-based testing solution into a scalable, web-based distributed system capable of orchestrating simultaneous hardware tests across geographically separated machines.

1. Architecture: Event-Driven Topology

Transformed the system from a synchronous 1-to-1 control model to an asynchronous 1-to-Many distributed topology.

  • Decoupled Communication: Implemented Apache Kafka as the central message backbone. The web server functions as a producer, broadcasting control commands to specific Kafka topics.
  • Distributed Consumption: Physical test stations act as consumers, subscribing to relevant topics to receive instructions and publishing telemetry data back to the cloud database.
  • Scalability: This architecture eliminated thread-blocking on the main server, allowing the system to scale to control hundreds of machines without performance degradation.

2. Resource Management: Concurrency & Channel Bundling

Developed the "Task Engine" to abstract physical hardware ports into logical test units.

  • Hardware Abstraction: Designed a scheduling system that allows users to bundle independent testing channels—residing on different physical machines—into a single logical "Task."
  • Concurrency Control: Implemented locking mechanisms and state management to ensure thread safety when multiple users attempt to reserve or control the same hardware channels simultaneously.
